# AGENTS.md

Гайд для AI-агента (Claude Code и др.), который продолжает разработку этого проекта.
Человеку-читателю — см. `README.md`; здесь фокус на том, как менять код безопасно.

## Что это

Мини-сервис: раз в день собирает истории с Hacker News, отбирает стоящее через
Claude Code CLI и постит дайджест в Telegram. Запускается по расписанию через
macOS `launchd`. Подробности пользовательского сценария — в `README.md`.

## Как запускать и проверять

```bash
uv run python -m hn_digest --dry-run   # прогон без отправки в Telegram (печать в консоль)
uv run python -m hn_digest             # боевой прогон
uv run pytest                          # юнит-тесты (без сети/claude/telegram)
```

`uv` сам поднимает окружение из `pyproject.toml`. Отдельно ставить зависимости не нужно.
**После любых изменений в логике гоняй `uv run pytest`.** Живые прогоны (`--dry-run`
и боевой) требуют сети, установленного `claude` и валидного логина — в CI/песочнице
они не пройдут, полагайся на pytest.

## Архитектура (поток данных)

```
sources.fetch_candidates()      # HN Algolia API → list[Story]
   → dedup.SeenStore.filter_new # выкидывает уже отправленные
   → curation.curate()          # claude -p → list[CuratedItem]
   → formatting.build_messages  # → list[str] (HTML, чанки ≤ 4096)
   → telegram.TelegramClient    # отправка
   → dedup.SeenStore.mark_sent  # запоминаем отправленное
```

Всё это связывает `pipeline.run()`; `pipeline.main()` добавляет логирование и
уведомление о падении.

## Карта модулей (`hn_digest/`)

| Модуль             | Ответственность                                                        |
|--------------------|------------------------------------------------------------------------|
| `config.py`        | `Settings` (dataclass) + загрузка `.env`. Единственный источник настроек.|
| `models.py`        | `Story` (кандидат) и `CuratedItem` (после Claude). Чистые dataclass'ы.  |
| `sources.py`       | Fetch с Hacker News (Algolia), пороги, дедуп внутри выборки.            |
| `dedup.py`         | `SeenStore` — персистентные антидубли (`state/seen.json`), TTL-прунинг.  |
| `curation.py`      | Вызов `claude -p`, промпт, парсинг JSON-ответа модели.                   |
| `formatting.py`    | Сборка Telegram-сообщений: HTML-экранирование, чанкинг под лимит.        |
| `telegram.py`      | `TelegramClient`: `send`, `send_all`, `send_error`.                      |
| `pipeline.py`      | Оркестрация + обработка падений + уведомление.                          |
| `logging_setup.py` | Единая настройка логов в stderr.                                        |
| `__main__.py`      | Точка входа `python -m hn_digest`.                                       |

## Конвенции

- Python ≥ 3.10, `from __future__ import annotations` во всех модулях.
- Тип-хинты обязательны для публичных функций; dataclass'ы со `slots=True`.
- Логирование через `logging_setup.get_logger(__name__)`, а не `print`
  (исключение — вывод дайджеста в `--dry-run`).
- Никаких зависимостей ради мелочей: сейчас в проде только `httpx`. Прежде чем
  добавлять пакет — взвесь, не решается ли задача стдлибом.
- Секреты только в `.env` (в git не коммитятся). Никогда не хардкодь токены в коде.
- Ошибки конфигурации/окружения — через `raise SystemExit("понятное сообщение")`.

## Подводные камни (важно)

- **`claude -p` и авторизация.** Курация идёт через подписку Claude Code (Max),
  НЕ через Anthropic API. Если в окружении есть битый `ANTHROPIC_API_KEY`, CLI
  падает с `401`. launchd наследует окружение логина — держи профиль чистым.
- **`--dry-run` не помечает отправленным.** `mark_sent` вызывается только после
  реальной доставки. Это осознанно: сухой прогон не должен «съедать» истории.
- **Помечаем только опубликованное**, а не всех кандидатов. Отклонённые Claude
  истории могут всплыть снова завтра (если наберут вистов) — это by design.
- **Лимит Telegram 4096.** Не собирай сообщения вручную — используй
  `formatting.build_messages`, там чанкинг с запасом (`SAFE_CHUNK`).
- **Всегда экранируй пользовательский текст** через `html.escape` перед вставкой
  в HTML-сообщение (заголовки/summary приходят от модели и внешних источников).
- **Песочница vs Mac.** Из некоторых сред нет прямого доступа к `api.telegram.org`
  и HN. Разрабатывай логику под pytest, живые прогоны — на реальной машине.

## Точки расширения (частые задачи)

- **Новый источник** (Lobsters, GitHub Trending, Product Hunt): добавь функцию в
  `sources.py`, верни `list[Story]`, слей с остальными до `dedup`. Модель `Story`
  универсальна — новые поля добавляй туда.
- **Другой формат вывода**: правь `formatting.py`, не трогая пайплайн.
- **Другой мессенджер**: сделай клиент рядом с `telegram.py` с тем же интерфейсом
  (`send_all`, `send_error`), подмени в `pipeline.run`.
- **Тюнинг отбора**: промпт в `curation.CURATION_PROMPT`; пороги — в `config`/`.env`.
- **Расписание**: `com.korney.hndigest.plist` (`StartCalendarInterval`).

При добавлении фичи: обнови модуль → добавь тест в `tests/test_core.py` →
`uv run pytest` → при необходимости обнови `README.md` и этот файл.

## Чего не делать

- Не коммитить `.env`, `state/`, логи (уже в `.gitignore`).
- Не переносить курацию на платный Anthropic API без явной просьбы владельца
  (смысл проекта — уложиться в подписку Max).
- Не удалять/переименовывать поля `Story`/`CuratedItem` без правки `formatting`
  и `curation` (они завязаны на имена полей из JSON модели).
